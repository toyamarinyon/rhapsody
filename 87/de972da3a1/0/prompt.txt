目的:
- admin-only の refresh probe endpoint を追加してください。
- mediator/proxy と同じ OAuth refresh request shape で `https://auth.openai.com/oauth/token` を叩き、refresh token が本当に使えるかを検証します。

想定 route:
`POST /api/v1/admin/codex-chatgpt-credentials/refresh-probe`

期待仕様:
1. `requireAdminAuth` 必須。
2. DB credential を `loadMediatorCredentialState()` で読む。
3. missing credential なら safe な 500/400 を返す。
4. `auth.openai.com/oauth/token` に POST:
   - `content-type: application/json`

レビューしました。実装の方向は良いですが、1点修正してください。

重大ではないですが安全上重要:
- `persist:true` のとき、現在の実装だと upstream が 400/401 でも body が JSON object なら `updateMediatorCredentialsFromOAuthResponse(state, parsedBody)` に進み得ます。
- failure body が `{ error: ..., refresh_token: ... }` のような形だった場合に DB を不正に更新する余地があります。

修正:
- DB 更新は `persist && upstreamResponse.ok` のときだけにしてください。
- upstream failure + `persist:true` でも `persisted:false` で、update が呼ばれないテストを追加してください。

追加でテスト可能なら:
- refresh request body に `client_id`, `grant_type: refresh_token`, `refresh_token` が入っていることを確認してください。ただし response/body snapshot には token 値を出さないでください。

修正後、biome/vitest/typecheck 結果をください。
