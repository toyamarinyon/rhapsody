`fetchIssueComments` が `octokit.rest.issues.listComments` を1回だけ呼んでいて、デフォルトページだけになっています。scheduler tick で最新の human reply を拾う用途なので、コメント数が30件を超えた issue で後続コメントを見落とす可能性があります。

修正方針:
- `fetchIssueComments` を pagination 対応にしてください。
- 既存の repo style に合わせるなら、`per_page: 100`, `page` loop + `response.headers.link?.includes('rel="next"')` でOKです。
- Octokit の `paginate` を使うなら injection type と test がやや複雑になるので、明示 page loop が無難です。
- 各 request に headers を維持してください。
- Test で2ページ目も取得するケースを追加してください。
