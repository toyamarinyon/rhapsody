Good skeleton. Please make these final safety/output fixes:

1. Remove support for `--root-password <value>` and `--root-password=<value>`. Passing a secret on the command line can leak via shell history/process lists. The helper should only read `ROOT_PASSWORD` from process env first, then `.env.local`; `--use-root-password` is the explicit opt-in to send it.
   - Unsupported `--root-password` should return JSON-only unsupported args.
   - Update nextActions/supported args text accordingly.
   - Update Facts so rootPasswordSource is only `process | .env.local | missing` with no CLI mapping.

2. Fix `.agents/skills/setup-rhapsody/SKILL.md` smoke-test bullet list:
   - indentation should match the surrounding numbered list style.
   - remove duplicate `trigger or wait for scheduler tick` line.

3. In the script, if the base URL is unreachable, nextActions should not say "Proceed to manual scheduler...". Blocked output should only tell the operator to fix blockers/re-run.
