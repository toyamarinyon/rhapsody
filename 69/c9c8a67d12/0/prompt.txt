実装をお願いします。ユーザーの明示的依頼により、intake proxy context の base64url 化をあなたに委譲します。こちらはレビュー・PR・merge・動作確認を担当します。既存変更を勝手に revert しないでください。

背景:
- #72 で ChatGPT OAuth refresh は mediator 固定 client_id を使うようにしましたが、#62 の intake classifier はまだ `Your access token could not be refreshed` で失敗しています。
- backend-probe / refresh-probe は production DB credential で成功しています。
- 次の有力仮説は、intake classifier が proxy URL の `{attemptId}` 相当に `workItemId` (`github_issue:toyamarinyon/rhapsody#62`) を入れていて、`/` を含む値が catch-all route 上で安全でないことです。
- 現在 `encodeURIComponent` はしていますが、Vercel/Next 側で `%2F` が path separator として扱われる可能性があるため、path-safe な base64url にしたいです。

目的:
- intake classifier 用 proxy context の workItemId を base64url 化して path segment に入れる。
- proxy route 側で base64url を decode して `listWorkItemGraph(decodedWorkItemId)` の active intake worker run check に使う。
- builder の通常 run/attempt path は壊さない。

対象候補:
- `lib/workers/intake-classifier-sandbox.ts`
- `app/api/internal/codex-chatgpt-proxy/[...path]/route.ts`
- `app/api/internal/codex-chatgpt-proxy/[...path]/route.test.ts`
- `lib/workers/intake-curator.test.ts` など関連テスト

期待実装:
1. `intake-classifier-sandbox` 側:
   - `workItemId` を `Buffer.from(workItemId, "utf8").toString("base64url")` などで base64url encode。
   - proxy URL は `/api/internal/codex-chatgpt-proxy/runs/{workerRunId}/attempts/{encodedWorkItemContext}` のように slash-free な segment にする。
   - 既存の `sanitizePathSegment` / `encodeURIComponent` 方式は base64url に置き換え。

2. proxy route 側:
   - active context check で、通常 builder attempt id と intake workItemId context を区別できるようにする。
   - 最小変更なら、`attemptId` をまず raw/URL decode し、その値で `getRunDetail` を試す。run が見つからない場合に base64url decode を試して workItemId として扱う。
   - base64url decode 失敗時は false。
   - decode 後の `graph.workItemId === decodedWorkItemId` は必ず維持。

3. テスト:
   - intake sandbox runner の codexProxyUrl が `%2F` や raw slash を含まず、base64url segment を使うこと。
   - proxy active check が base64url workItemId を decode して `intake_curator` running workerRun を許可すること。
   - mismatched base64url workItemId は拒否すること。
   - builder run/attempt context の既存テストは通ること。
