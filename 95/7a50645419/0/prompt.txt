# Issue Status Comment 実装計画

## Summary

Rhapsody の issue 上の `eyes` reaction を廃止し、代わりに **1つの管理コメント**を作成・更新する。コメントは Vercel の PR コメント風に、現在状態、dashboard/run/PR へのリンク、heartbeat、失敗時の次アクションを一元表示する。

Plan Mode のため、このターンでは実装は行わない。実装フェーズでは `builder` サブエージェントに以下の仕様を渡し、親エージェントが差分・テスト・表示内容をレビューする。

## Key Changes

- GitHub issue comment API を拡張する
  - `lib/github/issues.ts` に `updateIssueComment({ owner, repository, commentId, body })` を追加する。
  - Octokit の `issues.updateComment` を使う。
  - 既存の `GitHubIssueCommentError` を再利用する。

- `eyes` reaction を管理コメント upsert に置き換える
  - `lib/workers/intake-curator/index.ts` の `addIntakeEyesReaction` 相当の責務を変更する。
  - intake 実行開始時に `issue_status_comment` を作成または更新する。
  - reaction 失敗イベントは comment upsert 失敗イベントに置き換える。

- 管理コメント artifact を追加する
  - artifact kind として `issue_status_comment` を使う。
  - `externalId` に GitHub comment id、`externalUrl` に comment URL を保存する。
  - コメント本文末尾に HTML marker を含める。
    ```md
    <!-- rhapsody:issue-status-comment workItemId="..." -->
    ```
  - artifact がない場合は issue comments から marker を探して再利用する。

- コメント本文 renderer を追加する
  - state store / worker graph / latest run / latest artifact から Markdown を生成する。
  - 表示項目は以下に固定する。
    - status: queued / intake / in progress / PR ready / human review / completed / failed
    - current step
    - updated time
    - dashboard link
    - latest run link
    - pull request link if present
    - last heartbeat if present
    - latest meaningful event or failure point if present
  - 各 worker は自由文を持たず、保存済み状態から renderer が本文を再生成する。

- 更新タイミング
  - v1 では少なくとも以下で upsert する。
    - intake 開始時
    - intake classification 完了時
    - builder/runner が PR artifact を作った後
    - repair blocked comment 相当の状態になった時
    - scheduler が completed / failed / human review に進めた時
  - heartbeat callback では DB 側の heartbeat は維持し、GitHub comment 更新は「5分以上経過」または「状態変化あり」の場合だけにする。

## Comment Shape

例:

```md
Rhapsody is working on this issue.

| Status | Current step | Updated |
| --- | --- | --- |
| In progress | Running builder in Vercel Sandbox | May 28, 2026 10:42 UTC |

| Links | |
| --- | --- |
| Dashboard | [View work item](...) |
| Latest run | [wrun_...](...) |
| Pull request | [#123](...) |

Last heartbeat: May 28, 2026 10:41 UTC  
Latest event: Codex runner is still active.

<!-- rhapsody:issue-status-comment workItemId="..." -->
```

失敗時は冒頭を `Rhapsody needs attention.` にし、`Failure point` と `Next action` を含める。完了時は `Rhapsody completed this issue.` にする。

## Test Plan

- `lib/github/issues.test.ts`
  - `updateIssueComment` が正しい Octokit API と headers を呼ぶ。
  - GitHub API エラー時に `GitHubIssueCommentError` を投げる。

- intake curator tests
  - buildable item で eyes reaction が呼ばれない。
  - intake 開始時に `issue_status_comment` が作成される。
  - 既存 artifact がある場合は新規作成せず update する。
  - marker 付き既存コメントがある場合は再利用する。
  - comment upsert 失敗時は warn event を記録し、intake 自体は継続する。

- renderer tests
  - running / PR ready / completed / failed / stale heartbeat の Markdown が期待通り。
  - PR artifact、dashboard URL、heartbeat がない場合も壊れない。
  - HTML marker が常に含まれる。

- scheduler/runner integration-adjacent tests
  - PR artifact 作成後に status comment が PR link 付きで更新される。
  - repair blocked / human review / completed で本文が更新される。
